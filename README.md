# shared

Cross-module content for the SomaSafe monorepo: things more than one of
`application/`, `backend/` and `firmware/` needs, so they can't live inside just one of
them.

- **`docs/`** — topic docs for things that would otherwise be duplicated across the
  per-module READMEs (architecture, security, protocols, versioning semantics, ...). Each
  module's README covers what's specific to it and links here for shared background.
- **`dataset.proto`** — the capture-import protobuf, compiled independently by each
  consumer (`backend/scripts/export_subject_data.py` writes it, the Android app reads it).
- **`make_keys.sh`** — generates the server's ECDSA P-256 signing keypair (see
  [docs/model-signing.md](docs/model-signing.md)).
- **`gen/`** — build output from one project consumed by another: the dataset and
  exported models from the backend, the factory NVS image (incl. keys) from the firmware.
  Gitignored; regenerated by each project's own scripts.

## How it's linked

Every module has a convenience `make shared` target: it links `../shared` when run from
inside this monorepo layout, or, when that path doesn't exist, clones
[`memoria-somasafe-shared`](https://github.com/papyroplastics/memoria-somasafe-shared.git)
into `shared/` (gitignored) so the module builds standalone outside the monorepo too.
